## Quick Start

### Backend (Person 2)
```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload       # Starts on http://localhost:8000
```
API runs in DUMMY MODE until model.joblib exists.
Test: http://localhost:8000/health

### Frontend (Person 3)
```bash
cd frontend
npm install
cp .env.local.example .env.local
npm run dev                     # Starts on http://localhost:3000
```

### Model Training (Person 1)
```bash
cd backend
# Copy MSCAD.csv into /backend first
python train.py
# Outputs: model.joblib, label_encoder.joblib, feature_list.json, shap_background.npy
```

## Deploy

### Backend в†’ Railway
1. Push `/backend` to GitHub
2. railway.app в†’ New Project в†’ Deploy from GitHub
3. Select `/backend` as root directory
4. Railway auto-detects nixpacks.toml
5. Copy the Railway URL в†’ paste into frontend `.env.local`

### Frontend в†’ Vercel
1. Push `/frontend` to GitHub
2. vercel.com в†’ New Project в†’ Import repo
3. Set environment variable: `NEXT_PUBLIC_API_URL=<railway-url>`
4. Deploy
